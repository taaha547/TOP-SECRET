Core Web Vitals

Cumilative Layout Shift

Refers to how much the page shifts as elements load.
We want to minimize this to ensure a stable and user-friendly experience.
A CLS score of less than 0.1 is considered good.
Imagine this that you were pressing cancel order but the button shifted and you ended up pressing confirm order instead.
This is a bad user experience and we want to avoid this.

Larget Contextual paint

The way we focus on things is an interesting science in itself. We have a mental priority of what to notice first
LCP measures how long it takes for the largest content element to become visible within the viewport.
A good LCP score is 2.5 seconds or less.
This is important because it directly impacts user perception of how quickly a page loads.
If the main content takes too long to load, users may perceive the site as slow and unresponsive.
To improve LCP, we can optimize images, use efficient coding practices, and leverage browser caching.
Say for example you open a blog and the image takes too long to load, you might think the site is slow and leave.

First Input Delay

Measures the time from when a user first interacts with a page to the time when the browser is able to respond to that interaction.
Basically how quickly the site responds to user input.
A good FID score is less than 100 milliseconds.
This is crucial for user engagement, as delays in response can lead to frustration and abandonment of the site.
To improve FID, we can minimize JavaScript execution time, break up long tasks, and optimize event listeners.
For example, if you click a button and nothing happens for a few seconds, you might get frustrated and leave the site.

Lighthouse Report

Lighthouse is an open-source, automated tool for improving the quality of web pages. It provides audits for performance, accessibility, SEO, and more.
You can run Lighthouse in Chrome DevTools, from the command line, or as a Node module.
It generates a report that highlights areas for improvement and provides actionable insights to enhance the overall user experience.
For example, it might suggest optimizing images, reducing unused CSS, or improving accessibility features.
